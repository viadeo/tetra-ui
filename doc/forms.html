Placeholders

This is a polyfill to simulate the placeholder="" attribute for all browsers on <input> or <textarea> elements.

Call the JavaScript component at the end of your HTML page, for example just before closing the <body> tag:

<script src="/tetra-ui/dist/js/placeholder.js"></script>

Then the code for the input field is:

<span class="placeholding">
  <input type="text" />
  <label class="placeholder">I’m a placeholder</label>
</span>

<span class="field-helper field-helper-bottom">

To display the field helper on left:

<span class="field-helper field-helper-left">

<span class="field-helper field-helper-flow">

Inline form

Inputs are block elements by default. They are inline-block with .form-inline and .form-horizontal.

<form class="well form-inline">
<input type="text" class="input-small" placeholder="Email">
<input type="password" class="input-small" placeholder="Password">
<label class="checkbox">
<input type="checkbox"> Remember me
</label>
<button type="submit" class="btn btn-default">Sign in</button>
</form>

Horizontal forms

Here are the supported form elements:

  • text inputs (text, password, email, etc)
  • checkbox
  • radio
  • select
  • multiple select
  • file input
  • textarea

In addition to freeform text, any HTML5 text-based input appears like so.

Example markup

Given the above example form layout, here's the markup associated with the first input and control group. The .control-group, .control-label, and .controls classes are all required for styling.

<form class="form-horizontal">

    <input type="text" class="input-xlarge" id="input01">
    <p class="help-block">Supporting help text</p>
  </div>
</div>
</fieldset>
</form>

Extending form controls

To add help text for your form inputs, include inline help text with <span class="help-inline"> or a help text block with <p class="help-block"> after the input element.

You may also use static classes that don't map to the grid, adapt to the responsive CSS styles, or account for varying types of controls (e.g., <input> vs. <select>).

Note: Labels surround all the options for much larger click areas and a more usable form.

Custom radio & checkboxes

Only works in modern browsers, falls back to standard radio buttons in legacy browsers. These custom radio buttons are pure CSS3, no JavaScript required.

Just add a .custom-radio or .custom-checkbox class to your existing <label class="radio"> or <label class="checkbox">, and just after your <input /> field add an empty <span class="custom-radio-element"> or <span class="custom-checkbox-element">. 

<form class="core-form form-horizontal">
  <div class="control-group">

    <label class="control-label">Custom radio buttons</label>
    <div class="controls">

      <label class="radio inline custom-radio"><!-- Add the inline class for horizontal -->
        <input type="radio" name="customRadioButtons1" checked="checked" />
        <span class="custom-radio-element"></span><!-- This must be just before your text -->
        Option one
      </label>

      <label class="radio inline custom-radio">
        <input type="radio" name="customRadioButtons1" />
        <span class="custom-radio-element"></span>
        Option two
      </label>

    </div>

    <label class="control-label">Custom checkboxes</label>
    <div class="controls">

      <label class="checkbox inline custom-checkbox"><!-- Add the inline class for horizontal -->
        <input type="checkbox" checked="checked" />
        <span class="custom-checkbox-element"></span><!-- This must be just before your text -->
        Option one
      </label>

      <label class="checkbox inline custom-checkbox">
        <input type="checkbox" />
        <span class="custom-checkbox-element"></span>
        Option two
      </label>

    </div>

  </div>
</form>

Asynchronous files upload

Use HTML5 Form Data when possible and fallback to hidden iframe in “less capable” browsers.

Just add a .form-async-upload class to your form.
You also have to specify an id for it in order to receive the callbacks events correctly. 

<form class="core-form form-horizontal form-async-upload"
      id="upload_file" enctype="multipart/form-data" action="/upload" method="POST">
   ...
</form>

Events listened and triggered by the asynchronous file upload controller.

Events listened by the controller Description
page.notify('file_upload: submit', selector); Submit the form identified by the css 3 selector (exemple : "#myForm")
Events triggered by the controller Description
page.notify("file_upload: submitted", data); The file upload controller sends this event when the form is triggered. The variable data.id contains the CSS id of the form (in order for you to identify wich form was submitted if several are present on the page and take actions on it like display a loading state).
page.notify("file_upload: response", data); The file upload controller sends this event when the form is successfully submitted.
The variable data.resp contains the JSON response sent by the server and data.id the CSS id of the form.
Important note : the server response has to be sent in text/html or text/plain content-type in order to be handled by Internet Explorer, otherwise it will fail.
