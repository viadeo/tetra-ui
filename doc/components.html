Flipswitch

The flipswitch component maps a Tetra.js component to two native hidden radio buttons, the first one to yes and the second to no.

When yes is selected, the container has a flipswitch-yes class and the “yes” radio button is checked.

When no is selected, the container has a flipswitch-no class and the “no” radio button is checked.

<div class="flipswitch flipswitch-yes bootnode" id="flipswitch-sample-1"
  data-comp="flipswitch"
  data-event="click">

  <label class="flipswitch-lbl-yes">
    <input type="radio" name="flipswitch-sample-1" checked="checked" /> Yes
  </label>

  <span class="flipswitch-sliders">
    <span class="flipswitch-outer">
      <span class="flipswitch-inner">
        <span class="flipswitch-btn-no"></span>
        <span class="flipswitch-btn-yes"></span>
      </span>
    </span>
    <span class="flipswitch-slider"></span>
  </span>

  <label class="flipswitch-lbl-no">
    <input type="radio" name="flipswitch-sample-1" /> No
  </label>
</div>

On the JavaScript part, the component fires this event, and you can set the component state from a third party controller with the same function.

page.notify('flipswitch: set state', {
  id: 'flipswitch-sample-1', /* the DOM id of the toggle button */
  state: false /* true or false, the new state of the component */
});

Toggle buttons

The toggle button maps a Tetra.js component to a native hidden checkbox.

When active, the container has a btn-toggle-checked class and the hidden checkbox is checked.

When inactive, the btn-toggle-checked class is removed and the hidden checkbox is unchecked.

<label
  class="btn-toggle bootnode btn-toggle-checked"
  data-comp="toggle"
  data-event="click"
  id="btn-toggle-sample-1">

  <input type="checkbox" checked="checked" />
  <span class="btn-toggle-check">
    <i class="vicon">v</i>
  </span>

  Toggle button (default on)
</label>
<label
  class="btn-toggle bootnode"
  data-comp="toggle"
  data-event="click"
  id="btn-toggle-sample-2">

  <input type="checkbox" />
  <span class="btn-toggle-check">
    <i class="vicon">v</i>
  </span>

  Toggle button (default off)
</label>

Events triggered and listened by the component.

Triggered 
page.notify('toggle: state changed', {
  id: data.id, /* the DOM id of the toggle button */
  state: data.state /* true or false, the new state of the toggle */
});

This event is triggered when the state of the toggle button has changed.
Listened 
page.notify('toggle: set state', {
  id: data.id, /* the DOM id of the toggle button */
  state: data.state /* true or false, the value to be set */
});

This event sets the toggle button to a new state.

 
page.notify('toggle: invert state', {
  id: data.id /* the DOM id of the toggle button */
});

This event inverts the state of the toggle button.

Autocomplete

A generic autocomplete component with Ajax calls and templating.

The first autocomplete component sample calls this JSON response:
autocomplete-sample-1.json

And the second sample:
autocomplete-sample-2.json

The data-typing-delay="" attribute (not required) sets the delay in milliseconds before sending the server-side request.

The data-min-length="" attribute sets the minimum number of characters before autocompleting. 

      <div class="autocomplete bootnode" id="autocomplete-sample-1"
  data-comp="autocomplete"
  data-event="click"
  data-min-length="1"
  data-typing-delay="500"
  data-template-id="autocomplete_sample_1"
  data-url="/tetra-ui/doc/xhr/autocomplete-sample-1.json">

  <input type="text" class="input-large" />

  <ul class="autocomplete-menu"></ul>

  <script id="tmpl_autocomplete_sample_1" type="text/html">
    {% for (var i in data) { %}
      <li data-value="{%= data[i].fullName %}" data-ref="{%= data[i].ref %}">
        {%= data[i].fullName %} - <strong>{%= data[i].headline %}</strong>
      </li>
    {% } %}
  </script>
</div>
